## هل صمت وكيلك الذكي من قبل؟

رصيد API نفد. DeepSeek تحت حد المعدل. OpenAI تعطل. وكيلك كان عند 80% — ثم صمت.

المستخدمون لا يفهمون البنية التحتية. يعرفون فقط "تعطل مرة أخرى". ثلاث مرات وقد رحلوا.

## ماذا يفعل Zero Token

ليس "خطة احتياطية". ليس "وضع مخفّض". إنه **تسلم غير مرئي**.

في اللحظة التي يموت فيها نموذجك الرئيسي، 13 نموذج LLM مجاني يعملون بالفعل. نفس الشخصية. نفس القرارات. نفس الصوت. لا أحد يلاحظ التبديل.

**$0. صفر توقف. صفر اكتشاف.**

---

## سطرين. دقيقة واحدة.

```bash
openclaw skills install zero-token
bash ~/.openclaw/workspace/skills/zero-token/scripts/setup.sh
```

بوابة [Free-Way](https://github.com/GoDiao/Free-Way) تجمع 13+ مزود LLM مجاني خلف نقطة نهاية `/v1` واحدة. OpenClaw يستخدمها كبديل. تعطل النموذج الرئيسي؟ → تبديل تلقائي. وكيلك يستمر.

## شفافية كاملة

Zero Token لا يخفي التبديل أبداً. عند تفعيل fallback، وكيلك يعمل على مزودين مجانيين — وليس نموذجك الرئيسي. كل مزود لديه سياسة خصوصية واحتفاظ بالبيانات خاصة به. **راجع شروط كل مزود قبل إرسال بيانات حساسة.** سكريبت التثبيت يعرض بوضوح المزودين المضبطين. أنت تتحكم بالمفاتيح.

> 💡 هذا Failover شفاف، تتحكم به أنت. المفاتيح بيدك. أنت تختار المزودين.

---

## مجاني ≠ قمامة

المخرجات الخام للنماذج المجانية متوسطة. لكن مع DNA السلوك `::GENE{}` فوقها — نفس الشخصية، نفس القرارات، نفس النبرة. لا أحد يميز إن كان DeepSeek V4 Pro أو Llama 3.3 70B هو الذي يعمل.
